一、继承

1、什么是继承

extends关键字可以让一个类和另一个类建立起父子关系

子类写法：public class B extends A{}其中A类称为父类（基类或超类），B类称为子类（派生类）

子类能继承什么：子类能继承父类的非私有成员（成员变量、成员方法）（注：内存中存在≠继承）（静态成员不继承，绑定定义静态成员的类，所有父类子类共享同一块内存空间，子类可以写一个同名静态成员隐藏父类静态成员）

继承后对象的创建：子类的对象是由子类、父类共同完成的（因此虽然父类的非私有成员之外不能算作子类继承，但子类内存中仍然包含父类所有内容，只是不能直接访问）

2、权限修饰符

权限修饰符用来限制类中的成员（成员变量、成员方法、构造器）能够被访问的范围

private：只能本类访问
缺省：本类、同一个包中的类访问
protected：本类、同一个包中的类、子孙类中访问
public：任意位置访问

本类：都能访问
同包子类：缺省、protected、public能访问
同包不同类：缺省、protected、public能访问
不同包子类：protected、public能访问
不同包不同类：public能访问

3、继承的特点

单继承：java是单继承模式，一个类只能继承一个直接父类

多层继承：java不支持多继承（即子类继承不能继承自多个父类），但支持多层继承（即可以子类继承父类，父类也是继承自某个类的子类）

祖宗类：java中所有类都是继承自Object类的子类

直接继承或通过父类间接继承，不写继承就是默认继承Object

就近原则：先在子类局部范围找，找不到则去子类成员范围找，再找不到去父类成员范围找，如果还找不到则报错

如果子类父类中出现了重名的成员，优先使用子类成员，如果一定要在子类中使用父类的成员，可以通过super.父类成员来指定访问父类的成员

4、方法重写

实际开发中方法重写：声明不变，重新实现

当子类觉得父类的某个方法不好用，可以在子类中进行“方法重写”，覆盖父类的这个方法，要求：

方法名称、参数列表相同
在上方书写方法重写的校验注解@Override
访问权限必须>=父类方法的权限（public > protected > 缺省）
返回值必须与父类方法返回值类型一样或范围更小
父类方法不能为private或static（若父类方法为private，子类没继承，所以也不能重写；若父类方法为static，定义一个同名的方法属于方法隐藏）

4.1、方法重写常见应用场景

子类重写Object类的toString（）方法，以便于返回对象的内容

可以在idea中：右键->生成->toString（）

5、构造器

5.1、子类构造器

特点：子类的全部构造器都会先调用父类的构造器，再执行自己

调用父类构造器即super（有参/无参），不写则默认执行super（无参）

如果调用不了父类的无参数构造函数则必须在子类构造器第一行手写super（参数）调用父类的有参数构造器

5.2、兄弟构造器

任意类的构造器中，可以通过this（参数/无参）去调用该类的其他构造器

5.3、注意事项

super（）和this（）都必须写在构造器的第一行，并且两者不能同时出现

二、多态

1、什么是多态

多态是在继承/实现情况下的一种现象，表现为：对象多态、行为多态（而变量没有多态）

多态前提：有继承/实现关系，存在父类引用子类对象，存在方法重写

在父类对象引用子类对象时，并没有截断父类对象中没有的子类对象成员，只是父类对象不能访问

假设有父类 父类对象名 = new 多态子类构造器

那么当使用父类对象名调用方法时，父类若没定义方法，报错，不管子类是否定义都访问不到；父类若定义了，优先访问子类的方法
而使用父类对象名调用变量时，父类若没定义变量，报错，子类定义与否都访问不到；父类若写了，访问父类
总结：{对于方法：编译看左边，运行看右边}{对于变量：编译看左边，运行也看左边}

2、多态下的类型转换问题

自动类型转换：父类 变量名 = new 子类（）；
强制类型转换：子类 变量名 = （子类）父类变量；（前提是这个父类变量引用的是这个子类对象，其真实类型为子类对象）

强制类型转换好处：可以解决多态下不能调用子类独有功能的问题（前提是真实类型为子类或子类的后代）

2.1、强制类型转换的一个注意事项

假如父类为People，其子类为Teacher和Student，如下：

运行时，如果发现对象的真实类型与强转后的类型不同，就会报类型转换异常（ClassCastException）的错误出来（如上p的真实类型其实为子类Teacher，不是Student及其子类，不能强转Student）

强转前，建议使用instanceof关键字，判断当前对象的真实类型，再进行强转（对象 instanceof 目标类型判断对象是否是目标类型或目标类型的子类）

总结：子转父默认（看不到子类独有成员），父转子真实类型必须为子类或其后代

3、lombok技术

lombok技术可以为类自动添加getter、setter、无参构造器、toString方法等

可以使用@Data注解，然后下载lombok，再import lombok.Data；

若使用@Data注解，并不会自动生成有参数构造器，若想添加有参数构造器，可以使用@AllArgsConstructor注解自动添加一个包含所有参数的构造器

使用了@AllArgsConstructor注解就不会默认生成无参构造器了，此时若想无参构造器和有参构造器同时生成，需再添加一个@NoArgsConstructor注解一并生成无参构造器

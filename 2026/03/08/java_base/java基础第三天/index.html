一、数组

数组静态初始化：

一维数组：

类型[] 数组名 = new 类型[]{元素};
类型[] 数组名 = {元素};
二维数组：
类型[][] 数组名 = new 类型[][]{{元素}，{元素}};
类型[][] 数组名 = {{元素}，{元素}};

数组动态初始化：

一维数组：

类型[] 数组名 = new 类型[数组元素个数];
二维数组：
类型[] 数组名 = new 类型[数组元素个数][数组元素个数];

此时数组里面是默认值0、0.0、false、null

数组遍历：
使用下标遍历，并且可以使用数组名.length

二、面向对象编程入门

在同一个包里的不同类即使在不同的类文件里面也可以直接相互调用，无需导入

1、对象在计算机中存储区域

JVM虚拟机在运行java程序时将内存划分为：栈内存、堆内存、方法区

方法区：存放类

栈内存：存放方法的执行

堆内存：存放对象

栈内存中main方法里对应一个对象名的内存空间里存了堆内存里这个对象的地址

堆内存里对象这个内存空间里面存放了方法区里对应类的地址

2、类的构造器

结构：修饰符 方法名（）{}（无返回值，方法名必须与类名一样）

类默认带一个无参构造器，如果手动定义了有参数构造器，则默认的无参构造器没了，此时如果需要用无参构造器就需要手写一个

3、this关键字

this是一个变量，可以用在方法中，来拿到当前对象，哪个对象调用就拿到哪个对象

可用于解决成员变量和局部变量命名冲突的问题

4、封装

合理隐藏、合理暴露

如何隐藏：使用private修饰成员变量，就只能在本类中直接访问，其他任何地方都不能直接访问

如何暴露：使用public修饰的get和set等方法，在类外调用接口间接访问

5、实体类是什么

实体类是一种特殊类

实体类中要满足如下需求：

类中成员变量全部私有，并在类中提供public修饰的getter/setter方法
类中需要提供一个无参数构造器，有参数构造器可选

实体类仅仅用来保存数据，一般不做业务处理（不包含getter/setter），业务处理一般另起一个类操作此实体类，实现数据存储和业务处理相分离的开发方式

6、static关键字

static叫静态，可以修饰成员变量，成员方法

成员变量按照有无static修饰，分为两种：

静态变量（类变量）：有static修饰，属于类，在计算机里只有一份，被类的全部对象共享，推荐用类名.静态变量名访问
实例变量（对象的变量）：无static修饰，属于每个对象，用对象名.实例变量名访问

static修饰方法也分为静态方法和实例方法。
如果一个方法只是为了做一个功能且不需要直接访问对象的数据，这个方法可以直接定义成静态方法
如果这个方法是对象的行为，需要访问对象的数据，这个方法必须定义成实例方法

静态方法应用场景：编写工具类，减少编写重复代码

题外话：工具类没有创建对象的需求，建议将工具类的构造器进行私有

注意事项：

静态方法可以直接访问静态成员，但不可以直接访问实例成员，除非在静态方法内部构造对象并访问，此时称为间接访问
实例方法既可以直接访问静态成员，也可以直接访问实例成员
实例方法中可以出现this关键字，静态方法中不能出现this关键字

一、java加强

1、异常

异常代表程序出现的问题

1.1、java异常体系

  • Error：代表系统级别错误，属于严重问题（sun公司自己用）
  • Exception：异常，代表我们程序可能出现的问题

  • 运行时异常：RuntimeException及其子类，编译阶段不会报错，运行时出现的异常

  • 编译时异常：编译阶段就会出现报错

1.2、异常处理

  • 抛出异常（throws）：在方法上使用throws关键字，可以将方法内部出现的异常抛出去给调用者

  • 捕获异常（try…catch）：直接捕获程序出现的异常

    （建议直接抛Exception异常，捕获也直接捕获Exception异常，可以处理所有异常）

1.3、异常的作用

  • 异常是用来定位程序bug的关键信息
  • 可以作为方法内部的一种特殊返回值以便通知上层调用者，方法的执行问题

1.4、自定义异常

java无法为世界上全部问题提供异常类来代表，如果企业自己的某种问题想通过异常表示，以便用异常来管理该问题，那就需要自己来定义异常类了

自定义运行时异常：

  • 定义一个异常类继承RuntimeException
  • 重写构造器
  • 通过throw new 异常类（xxx）来创建异常对象并抛出

特点：编译阶段不报错，运行时才可能出现。提醒不属于激进型

自定义编译时异常：

  • 定义一个异常类继承Exception
  • 重写构造器
  • 通过throw new 异常类（xxx）创建异常对象并抛出

特点：编译阶段就报错，提醒比较激进

1.5、异常的处理

方案一（常用）：

底层异常层层往上抛出，最外层捕获异常，记录下异常信息，并响应适合用户观看的信息进行提示

方案二（少见）：

最外层捕获异常后，尝试重新修复

2、泛型

定义类、接口、方法时，同时声明了一个或多个类型变量（如：）称为泛型类、泛型接口、泛型方法，他们统称为泛型

泛型支持的类型：泛型不支持基本数据类型，只能支持对象类型（引用数据类型）

作用：泛型提供了在编译阶段约束所能操作的数据类型，并自动进行检查的能力。这样可以避免强制类型转换，及其可能出现的异常

泛型的本质：编译阶段把具体的数据类型传给类型变量，将所有类型变量替换成具体数据类型。编译阶段过后，泛型会被擦除，所有类型恢复成Object类型

注意：类型变量建议用大写的英文字母，常用E、T、K、V等（其中E一般代表元素类型，T一般代表返回值类型，K一般代表键类型，V一般代表值类型）

2.4、通配符、上下限

通配符就是” ? “，可以在使用泛型的时候代表一切类型；E、T、K、V是在定义泛型的时候使用

例如有一个父类car，其有两个子类BMW和BYD，虽然接收参数时可以使用car类型接收这三种类的对象，但如果用ArrayList存这三种类型，就不能使用ArrayList接收了，因为car是BMW和BYD的父类，但ArrayList和ArrayList和ArrayList没有任何关系，此时就可以使用通配符用ArrayList<?>接收三种类型了

泛型上限：

? extends 类型: ?

能接收的必须是这个类型或其子类

泛型下限：

? super 类型: ?

能接收的必须是这个类型或其父类

2.5、包装类

由于泛型不支持基本数据类型，因此我们可以使用包装类来间接使用基本数据类型

包装类就是把基本类型的数据包装成对象的类型

基本数据类型 对应的包装类
byte Byte
short Short
int Integer
long Long
char Character
float Float
double Double
boolean Boolean

自动装箱：基本数据类型可以自动转换为包装类型

  • 可以使用 包装类类型 对象名 = 数据

自动拆箱：包装类型可以自动转换为基本数据类型

  • 可以使用 基本数据类型 变量名 = 对应包装类对象

包装类具备的其他功能

  • 可以把基本类型的数据转换成字符串类型（用的少）

  • 可以把字符串类型的数值转换成数值本身对应的真实数据类型（用的多）

​ 基本数据类型 变量名 = 对应包装类.valueOf(字符串对象)

3、集合

3.1、集合体系结构

Collection：单列集合，每个元素只包含一个值

Map：双列集合，每个元素包含两个值（键值对）

常见体系结构：

接口Collection

接口List

实现类ArrayList

实现类LinkedList

接口Set

实现类HashSet

实现类LinkedHashSet

实现类TreeSet

Collection集合特点：

  • List系列集合：添加的元素是有序、可重复、有索引

  • Set系列集合：添加的元素是无序、不重复、无索引（其中LinkedHashSet有序，TreeSet按照大小默认升序排序）

3.2、Collection的常用功能

方法名 说明
public boolean add(E e) 把给定的对象添加到当前集合中
public void clear() 清空集合中所有的元素
public boolean remove(E e) 把给定的对象在当前集合中删除
public boolean contains(Object obj) 判断当前集合中是否包含给定的对象
public boolean isEmpty() 判断当前集合是否为空
public int size() 返回集合中元素的个数
public Object[] toArray() 把集合中的元素存储到数组中

3.3、Collection的遍历方式

3.3.1、迭代器遍历

迭代器是用来遍历集合的专用方式（数组没有迭代器），java中迭代器代表是Iterator

Collection集合获取迭代器的方法

方法名称 说明
Iterator iterator() 返回集合中的迭代器对象，该对象默认指向当前集合第一个元素

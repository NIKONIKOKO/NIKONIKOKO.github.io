一、面向对象高级

1、final关键字

final关键字可以修饰：类、方法、变量

修饰类：该类被称为最终类，特点是不能被继承了（final后跟class）

修饰方法：该方法被称为最终方法，特点是不能被重写了（final后跟返回类型）

修饰变量：该变量有且仅能被赋值一次（final后跟变量类型）

1.1、常量

使用了static final修饰的成员变量就被称为常量

作用：常用于记录系统的配置信息

命名规范：建议全部使用大写，多个单词使用下划线连接起来

使用常量记录系统配置信息优势：代码可读性好，可维护性好，并且程序编译后，常量会被“宏替换”替换成字面量，可以保证使用常量和直接使用字面量性能一样

1.2、final注意事项

final修饰基本类型的变量，变量存储的数据不能被改变

final修饰引用类型的变量，变量存储的地址不能被改变，但地址所指向对象的内容是可以被改变的

2、设计模式

设计模式：一个问题通常有n种解法，其中肯定有一种解法是最优的，这个最优的解法被人总结出来了，称为设计模式

设计模式有20多种，对应20多种软件开发中会遇到的问题

学习设计模式：明白能解决什么问题、知道怎么写

2.1、单例设计模式

作用：确保某个类只能创建一个对象

写法：

饿汉式单例（拿对象时，对象早就创建好了）

类的构造器私有（确保单例类对外不能创建更多对象，单例才有可能性）
定义一个静态类变量记住类的一个对象（定义时引用一个已经创建好的对象）
定义一个静态类方法，返回对象（不是静态方法依赖对象存在，而我们需要通过类调用方法）
懒汉式单例（拿对象时，才开始创建对象）
类的构造器私有
定义一个静态类变量用于存储对象（定义时不创建对象指向null）
提供一个静态类方法，返回对象（通过类名直接调用静态方法，静态对象已经创建过就返回同一个静态对象，没有创建就先创建再返回）

3、枚举类

枚举是一种特殊类

3.1、特点

枚举都是最终类，不可以被继承，枚举类都是继承java.lang.Enum类的
枚举类中的第一行，只能写枚举类的对象名称，且要用逗号隔开。这些名称，本质是常量，每个常量都是类型为此枚举类的一个实例对象
枚举类的构造器都是私有的（写不写都是私有），因此枚举类对外不能创建对象

3.2、注意事项

switch的case后面不能使用枚举类名.枚举变量名，应该直接使用枚举变量名，不然报错

4、抽象类

abstract关键字可以修饰类（abstract后跟class）、成员方法（abstract后跟返回类型），分别称为抽象类、抽象方法

4.1、特点以及注意事项

抽象类最主要特点：抽象类不能创建对象，仅作为一种特殊的父类，让子类继承并实现（不能创建对象但可以引用子类对象）
抽象类中不一定要有抽象方法，有抽象方法的类必须是抽象类
类有的成员（成员变量、方法、构造器）抽象类都可以有
抽象类里面如果写抽象方法，只能写方法的声明而不能写具体的方法体（普通方法可以写方法体）
一个类继承抽象类，必须重写完抽象类的全部抽象方法，否则这个类也必须定义成抽象类

4.2、模板方法设计模式

作用：

提供一个方法作为完成某类功能的模板，模板方法封装了每个实现步骤，但允许子类提供特定步骤的实现
提高代码的复用、并简化子类设计

写法：

1、定义一个抽象类
2、在里面定义2个方法

一个是模板方法：把共同的实现步骤放里面去（建议给模板方法加final关键字，防止被子类重写后滥用）
一个是抽象方法：不确定的实现步骤，交给具体的子类来完成

5.2、接口特点

接口不能创建对象，但可以引用实现类对象，然后调用实现类重写的接口里面定义的方法（接口里没定义的不能调用）

5.3、接口作用

接口是用来被类实现（implements）的，实现接口的类称为实现类，一个类可以同时实现多个接口

如：

好处：
弥补了类单继承的不足，一个类能同时实现多个接口，角色更多，功能更强
让程序可以面向接口编程，可以灵活方便的切换各种业务实现（更利于程序解耦合）

5.4、JDK8之后接口新增三种方法

5.4.1、默认方法

默认方法（普通实例方法），必须加default，默认使用public修饰

注意：只能使用接口的实现类对象调用（或者接口引用实现类对象来调用，本质还是通过实现类对象调用）

5.4.2、私有方法

JDK9之后新增的

私有的实例方法，使用private修饰，可以不加default

使用接口中其他实例方法(默认方法、私有方法)调用

5.4.3、静态方法

默认public修饰

只能使用当前接口名来直接调用（不允许实现类的类名以及实现类实例对象调用，接口的静态对象不被实现类继承，实现类看不到静态对象）

也可以使用默认方法、私有方法来间接调用

5.5、接口注意事项

接口与接口可以多继承，一个接口可以同时继承多个接口

一个接口继承多个接口，如果多个接口中存在方法签名冲突，则此时不支持多继承、多实现（此时讨论抽象方法）

方法签名完全一致则可以规范合并，支持多继承、多实现
返回值一个为父类一个为子类，协变返回，支持多继承、多实现
仅参数不同，方法重载，支持多继承、多实现
仅返回值不同，冲突，不支持多继承、多实现

一个类继承了父类，又同时实现了接口，如果父类中和接口中有同名的默认方法，实现类优先使用父类的（如果要调接口的默认方法，使用接口名.super.默认方法名（），此时并不是通过接口名调用，只是指定调用接口的方法，本质还是依赖实现类对象调用）

一个类实现了多个接口，如果多个接口中存在同名默认方法，可以不冲突，这个类重写该方法即可（此时讨论默认方法）

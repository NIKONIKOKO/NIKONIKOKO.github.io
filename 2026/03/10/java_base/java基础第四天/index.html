一、继承

1、什么是继承

extends关键字可以让一个类和另一个类建立起父子关系

子类写法：public class B extends A{}其中A类称为父类（基类或超类），B类称为子类（派生类）

子类能继承什么：子类能继承父类的非私有成员（成员变量、成员方法）（注：内存中存在≠继承）（静态成员不继承，绑定定义静态成员的类，所有父类子类共享同一块内存空间，子类可以写一个同名静态成员隐藏父类静态成员）

继承后对象的创建：子类的对象是由子类、父类共同完成的（因此虽然父类的非私有成员之外不能算作子类继承，但子类内存中仍然包含父类所有内容，只是不能直接访问）

2、权限修饰符

权限修饰符用来限制类中的成员（成员变量、成员方法、构造器）能够被访问的范围

private：只能本类访问
缺省：本类、同一个包中的类访问
protected：本类、同一个包中的类、子孙类中访问
public：任意位置访问

本类：都能访问
同包子类：缺省、protected、public能访问
同包不同类：缺省、protected、public能访问
不同包子类：protected、public能访问
不同包不同类：public能访问

3、继承的特点

单继承：java是单继承模式，一个类只能继承一个直接父类

多层继承：java不支持多继承（即子类继承不能继承自多个父类），但支持多层继承（即可以子类继承父类，父类也是继承自某个类的子类）

祖宗类：java中所有类都是继承自Object类的子类

直接继承或通过父类间接继承，不写继承就是默认继承Object

就近原则：先在子类局部范围找，找不到则去子类成员范围找，再找不到去父类成员范围找，如果还找不到则报错

如果子类父类中出现了重名的成员，优先使用子类成员，如果一定要在子类中使用父类的成员，可以通过super.父类成员来指定访问父类的成员

4、方法重写

实际开发中方法重写：声明不变，重新实现

当子类觉得父类的某个方法不好用，可以在子类中进行“方法重写”，覆盖父类的这个方法，要求：

方法名称、参数列表相同
在上方书写方法重写的校验注解@Override
访问权限必须>=父类方法的权限（public > protected > 缺省）
返回值必须与父类方法返回值类型一样或范围更小
父类方法不能为private或static（若父类方法为private，子类没继承，所以也不能重写；若父类方法为static，定义一个同名的方法属于方法隐藏）

4.1、方法重写常见应用场景

子类重写Object类的toString（）方法，以便于返回对象的内容

可以在idea中：右键->生成->toString（）

5、构造器

5.1、子类构造器

特点：子类的全部构造器都会先调用父类的构造器，再执行自己

调用父类构造器即super（有参/无参），不写则默认执行super（无参）

如果调用不了父类的无参数构造函数则必须在子类构造器第一行手写super（参数）调用父类的有参数构造器

5.2、兄弟构造器

任意类的构造器中，可以通过this（参数/无参）去调用该类的其他构造器

5.3、注意事项

super（）和this（）都必须写在构造器的第一行，并且两者不能同时出现

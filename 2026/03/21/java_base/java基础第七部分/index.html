一、java加强

1、异常

异常代表程序出现的问题

1.1、java异常体系

  • Error：代表系统级别错误，属于严重问题（sun公司自己用）
  • Exception：异常，代表我们程序可能出现的问题

  • 运行时异常：RuntimeException及其子类，编译阶段不会报错，运行时出现的异常

  • 编译时异常：编译阶段就会出现报错

1.2、异常处理

  • 抛出异常（throws）：在方法上使用throws关键字，可以将方法内部出现的异常抛出去给调用者

  • 捕获异常（try…catch）：直接捕获程序出现的异常

    （建议直接抛Exception异常，捕获也直接捕获Exception异常，可以处理所有异常）

1.3、异常的作用

  • 异常是用来定位程序bug的关键信息
  • 可以作为方法内部的一种特殊返回值以便通知上层调用者，方法的执行问题

1.4、自定义异常

java无法为世界上全部问题提供异常类来代表，如果企业自己的某种问题想通过异常表示，以便用异常来管理该问题，那就需要自己来定义异常类了

自定义运行时异常：

  • 定义一个异常类继承RuntimeException
  • 重写构造器
  • 通过throw new 异常类（xxx）来创建异常对象并抛出

特点：编译阶段不报错，运行时才可能出现。提醒不属于激进型

自定义编译时异常：

  • 定义一个异常类继承Exception
  • 重写构造器
  • 通过throw new 异常类（xxx）创建异常对象并抛出

特点：编译阶段就报错，提醒比较激进

1.5、异常的处理

方案一（常用）：

底层异常层层往上抛出，最外层捕获异常，记录下异常信息，并响应适合用户观看的信息进行提示

方案二（少见）：

最外层捕获异常后，尝试重新修复

2、泛型

定义类、接口、方法时，同时声明了一个或多个类型变量（如：）称为泛型类、泛型接口、泛型方法，他们统称为泛型

泛型支持的类型：泛型不支持基本数据类型，只能支持对象类型（引用数据类型）

作用：泛型提供了在编译阶段约束所能操作的数据类型，并自动进行检查的能力。这样可以避免强制类型转换，及其可能出现的异常

泛型的本质：编译阶段把具体的数据类型传给类型变量，将所有类型变量替换成具体数据类型。编译阶段过后，泛型会被擦除，所有类型恢复成Object类型

注意：类型变量建议用大写的英文字母，常用E、T、K、V等（其中E一般代表元素类型，T一般代表返回值类型，K一般代表键类型，V一般代表值类型）

2.4、通配符、上下限

通配符就是” ? “，可以在使用泛型的时候代表一切类型；E、T、K、V是在定义泛型的时候使用

例如有一个父类car，其有两个子类BMW和BYD，虽然接收参数时可以使用car类型接收这三种类的对象，但如果用ArrayList存这三种类型，就不能使用ArrayList接收了，因为car是BMW和BYD的父类，但ArrayList和ArrayList和ArrayList没有任何关系，此时就可以使用通配符用ArrayList<?>接收三种类型了

泛型上限：

? extends T

能接收的类型必须是这个T类型或其子类

泛型下限：

? super T

能接收的类型必须是这个T类型或其父类

3.5、并发修改异常问题

遍历集合的同时又存在增删集合元素的行为时可能出现业务异常，这种现象被称之为并发修改异常问题

例如：需要连续删两个，第一个删除之后第二个立马往前补位，但是已经遍历到下一位置，即需要删除的第二个的原本位置，此时漏删了第二个。这就是并发修改异常问题

Collection并发修改异常问题：

  • 方案一：迭代器遍历并删除默认也存在并发修改异常问题

可以解决，使用迭代器自己的方法来删除而不是用集合的方法来删除

  • 方案二、三：增强for循环和Lambda表达式遍历并删除也存在并发修改异常问题

都没有办法解决并发修改异常问题，底层都是迭代器遍历，但无法显式使用迭代器的方法删除，都只能使用集合的方法删除，所以都解决不了

结论：

  • 当集合有索引时，可以删完一个手动退一格，或者可以从后往前遍历并删除，都可以解决并发修改异常问题

  • 当集合没有索引时，只有迭代器遍历可以解决并发修改异常问题，其他两种遍历方法不行

3.6.1、ArrayList的底层原理

ArrayList底层是基于数组存储数据的

数组特点：

  • 数组中数据在内存中存储是连续的

  • 查询速度快（根据索引快速定位）

  • 增删数据效率低（需要前移或者扩容数组并拷贝）

3.6.2、LinkedList的底层原理

LinkedList底层是基于双链表存储数据的

链表特点：

  • 链表中的数据是一个一个独立结点组成的，结点在内存中不连续，每个结点包含数据值和下一个结点的地址

  • 查询慢，无论查询哪个数据都要从头开始找

  • 链表增删相对快

双链表独有特点：

  • 对首尾元素进行增删改查速度极快

LinkedList因为基于双链表，所以新增了很多首尾操作的特有方法

方法名称 说明
public void addFirst(E e) 在该列表开头插入指定的元素
public void addLast(E e) 将指定的元素追加到此列表的末尾
public E getFirst() 返回此列表中的第一个元素
public E getLast() 返回此列表中的最后一个元素
public E removeFirst() 从此列表中删除并返回第一个元素
public E removeLast() 从此列表中删除并返回最后一个元素

3.7.1、HashSet集合

哈希值：

  • 就是一个int类型的随机值，java中每个对象都有一个哈希值
  • java中所有对象，都可以调用Object类提供的hashCode方法，返回该对象自己的哈希值

​ public int hashCode(); 返回对象的哈希码值

对象哈希值的特点：

  • 同一个对象多次调用hashCode（）方法返回的哈希值是相同的
  • 不同的对象，他们的哈希值大概率不相等，但也有可能会相等（哈希碰撞）

哈希表

  • JDK8之前，哈希表 = 数组 + 链表
  • JDK8开始，哈希表 = 数组 + 链表 + 红黑树
  • 哈希表是一种增删改查数据，性能都较好的数据结构

HashSet集合是基于哈希表存储数据的

3.7.1.1、HashSet底层工作机制

JDK8之前：

  1. 创建一个默认长度16的数组，默认加载因子为0.75，数组名table，当存储数据总数超过16 * 0.75 = 12个时，扩容成2倍，重新计算哈希值分配位置
  2. 使用元素的哈希值对数组长度做运算计算出应存入的位置
  3. 判断当前位置是否为null，如果是null直接存入
  4. 如果不为null，表示有元素，则调用equals方法比较，相等则不存；不相等则存入数组
    • JDK8之前，新元素存入数组，占老元素位置，老元素挂下面
    • JDK8开始之后，新元素直接挂在老元素下面

3.7.1.2、HashSet集合去重复的机制

不同内容也可能hash到同一位置，所以想去重不能仅hashCode相同；此外，如果仅equals，可能相同内容的hashCode映射到不同位置，也无法实现去重。因此得到以下结论：

  • 如果希望Set集合认为两个内容一样的对象是重复的，必须重写对象的hashCode()和equals()方法，保证两个内容的hashCode相同且内容相同，这样才可以去重

  • 但是，每个元素都额外多了一个双链表的机制来记录它前后元素的位置

  • 并且LinkedHashSet也有头指针尾指针

3.7.3、TreeSet集合

底层是基于红黑树实现的排序

注意：

  • 对于数值类型：Integer、Double，默认按照数值本身的大小进行升序排序
  • 对于字符串类型：默认按照首字符的编号升序排序
  • 对于自定义类型如Student对象，TreeSet默认是无法直接排序的（因为TreeSet必须要排序，所以不写排序规则进程会直接崩溃）

TreeSet集合如果为自定义类型，必须要手写排序规则，有两种方案：

  • 方案一：

对象类实现一个Comparable比较接口，重写compareTo方法，指定大小比较规则

如果你认为调用compareTo的对象大于被比较对象，返回正整数；小于返回负整数；等于返回0。

如果返回了0，则会当成重复内容，会被去重

  • 方案二：

public TreeSet (Comparator c)集合自带比较器Comparator对象，指定比较规则

3.8、Map集合体系结构

Map：双列集合，每个元素包含两个值（键值对）

常见体系结构：

接口Map<K,V>

实现类HashMap<K,V>

实现类LinkedHashMap<K,V>

实现类TreeMap<K,V>

Map集合也叫“键值对集合”，格式：{key1 = value1，key2 = value2，key3 = value3，…}

Map集合的所有键是不允许重复的，但值可以重复，键和值是一一对应的，每一个键只能找到自己对应的值

Map集合特点（特点都由键决定；值是附属品，不做要求；键和值都可以是null）：

  • HashMap：无序、不重复、无索引（用的最多）
  • LinkedHashMap：有序、不重复、无索引
  • TreeMap：按照大小默认升序排序、不重复、无索引

3.12、Stream流

JDK8开始新增的一套API，可以用于简化操作集合或者数组的数据

优势：Stream流大量的结合了Lambda的语法风格来编程，功能强大，性能高效，代码简洁，可读性好

3.12.1、Stream流的使用步骤

数据源（集合/数组/…）

↓

1、获取Stream流（Stream流代表一条流水线，并能与数据源建立连接）

↓

2、调用流水线的各种方法，对数据进行处理、计算（过滤、排序、去重…）

↓

3、获取处理的结果，遍历、统计、收集到一个新集合中返回

↓

获取结果

3.12.2、获取Stream流

Stream流接口：

public interface Stream<T> ...{}

  • 获取集合的Stream流：

Collection封装了stream（）方法可以获取当前集合对象的Stream流：

​ default Stream<E> stream()

注意：Map集合可以通过keySet().stream()获取键流，也可以通过entrySet().stream()获取键值对流

  • 获取数组的Stream流：

Arrays类封装了stream(T[] array)方法可以获取当前数组的Stream流：

​ public static <T> Stream<T> stream(T[] array)

Stream类封装了of(T… values)方法可以获取当前接收数据的Stream流：

​ public static <T> Stream<T> of(T... values)（其中T…为可变参数，可接一个参数、多个参数、一个数组）

中间方法指的是调用完成后会返回新的Stream流，支持链式编程

终结方法指的是调用完成后，不会返回新Stream了，没法继续使用流了
